# OctoAcme Project Management Docs

This README is the central index for OctoAcme project management documentation. Use it as the starting point for onboarding, understanding how teams run projects, and finding the right process guide for each phase of delivery.

## Process Summary

OctoAcme uses a lightweight, iterative project management approach that moves work from initiation through planning, execution, release, and retrospective improvement. Teams align early on the problem to solve, the goals to achieve, and the success metrics that will show whether the work delivered value. Planning then turns approved work into a prioritized backlog with clear acceptance criteria, release milestones, and identified risks or dependencies. During execution, teams use regular cadences, project boards, pull requests, and testing practices to track progress and surface blockers quickly. Releases focus on readiness, verification, and communication, while retrospectives help teams capture learnings, improve the process, and continuously strengthen delivery over time.
